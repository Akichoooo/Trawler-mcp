参赛赛道：学习工作

Trawler-mcp

一个为 LLM / Agent 打造的可信网页爬取 MCP Server。把混乱的互联网变成 LLM 能直接读懂的干净 Markdown，并具备反爬降级、人工兜底与 SSRF 安全防御。

1. 创意介绍

想解决什么问题

当下 Agent IDE（如 TRAE、Cursor）和 RAG 系统需要大量实时网页数据，但现有爬虫工具普遍存在三个问题：反爬一碰就碎、返回内容噪声大、安全与并发不可控。LLM 拿到的是一堆 HTML 标签或失败提示，无法真正“看懂”网页。

灵感来源

在把 TRAE 当作 Agent 开发环境使用时，经常需要让 Agent 读取文档、GitHub Issue、论坛帖子。遇到 Cloudflare 或登录墙时，要么直接失败，要么需要手动复制粘贴，打断了 Agent 的自动流程。因此我们希望有一个“交给 Agent 就能放心用”的网页读取工具。

产品形态

Trawler-mcp 是一个基于 MCP（Model Context Protocol） 的网页爬取服务，提供 crawl_url、crawl_site 等工具。它输出干净的 Markdown，内置 patchright → Jina Reader → HITL 三档阶梯、账号库、SSRF 防御、审计日志与“手册学习”机制。

2. 核心流程

URL 输入规范化 + SSRF 初检
→
查手册cached_gear 跳档
→
三档阶梯patchright / Jina / HITL
→
Parser提取正文 Markdown
→
落盘审计raw + audit + seen
  • patchright 档：反检测 headless Chrome，注入账号 storage_state，可过 Cloudflare / 复杂 SPA。
  • Jina Reader 档：当 patchright 被 Cloudflare 拦截时，快速获取已渲染内容。
  • HITL 档：遇到登录/复杂验证码时，弹出浏览器等人过；成功后保存状态供后续复用。
  • SSRF 双检：入口 URL + patchright 最终 final_url 都会检查，命中直接阻断。

3. 目标用户及痛点

Agent / RAG 开发者

需要让 LLM 读取网页，但不想维护 Selenium/Playwright 一堆环境；希望一次接入 MCP 就能用。

科研人员 / 分析师

批量跟踪论文、论坛、竞品页面，现有工具遇到反爬就断，缺少自动降级和断点续爬。

企业知识库团队

需要把外部公开文档同步到内部系统，对安全（SSRF）、审计、账号隔离有硬性要求。

使用场景

竞品监控、论文采集、电商价格跟踪、企业内部知识库同步、Agent 实时联网查询。

4. 价值与意义

  • 效率提升：把“让 LLM 能读网页”从数小时自建工作缩短为一条 MCP 命令；多档阶梯自动选择成本最低的成功路径，避免一上来就开浏览器。
  • 安全可控：通过 SSRF 防御、账号隔离、审计日志，让 Agent 联网行为可审计、可阻断，降低自动化工具被滥用风险。
  • 生态兼容：基于 MCP 协议，可无缝接入 TRAE、Cursor、Claude Desktop 等任何支持 MCP 的 Agent IDE。
  • 开放演进：“手册学习”机制让系统根据历史成功率自动选择起手档位，越用越省成本。

5. 技术栈与当前进展

  • 后端：Python + FastMCP + SQLite（WAL）+ asyncio
  • 浏览器：patchright / playwright，支持 storage_state 注入
  • 正文提取：trafilatura / readability-lxml / html2text 多 Parser 兜底
  • 安全：ipaddress + socket.getaddrinfo 双栈 SSRF 拦截，路径穿越白名单
  • 当前状态：已完成 crawl_url 单页流水线和 crawl_site 同域蜘蛛爬，含单元测试 19 项全绿。
